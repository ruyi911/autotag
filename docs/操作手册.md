# 操作手册

## 1. 环境准备

```bash
cd /Users/momo/Desktop/autotag
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
```

## 2. 首次部署

### 2.1 直接跑某日完整链路（有远端抓取）

```bash
bash scripts/run_daily.sh 2026-03-14
```

### 2.2 仅使用本地 `data/initial_csv/`

当前 `scripts/init_backfill.sh` 只做文件搬运到 `data/raw_files/dt=<date>/...`，不会生成 manifest，也不会入库。

推荐流程：

```bash
# 1) 从 initial_csv 生成该 dt 的 manifest + 归档（不走远端 API）
PYTHONPATH=src .venv/bin/python -m autotag.ingest.downloader \
  --dt 2026-03-14 \
  --include-initial \
  --sources user,recharge,withdraw,bet,bonus

# 2) 跑后续 load/model/publish（跳过下载）
bash scripts/run_daily.sh 2026-03-14 --skip-download
```

如果仅需要搬运文件，可单独使用：

```bash
bash scripts/init_backfill.sh 2026-03-14
```

## 3. 日常运行

```bash
# 默认 dt = 印度时区昨天
bash scripts/run_daily.sh

# 指定 dt
bash scripts/run_daily.sh 2026-03-14
```

常用参数：

```bash
bash scripts/run_daily.sh 2026-03-14 --skip-download
bash scripts/run_daily.sh 2026-03-14 --sources user,recharge,withdraw
bash scripts/run_daily.sh 2026-03-14 --mode replay
bash scripts/run_daily.sh 2026-03-14 --mode realtime
bash scripts/run_daily.sh 2026-03-14 --no-publish
```

## 4. 区间补数

```bash
bash scripts/backfill.sh --start-date 2026-03-01 --end-date 2026-03-07
```

可选参数：

```bash
bash scripts/backfill.sh --start-date 2026-03-01 --end-date 2026-03-07 --only-sources user,recharge
bash scripts/backfill.sh --start-date 2026-03-01 --end-date 2026-03-07 --skip-download
bash scripts/backfill.sh --end-date 2026-03-07 --from-last-success
```

## 5. 周期回放

```bash
bash scripts/weekly_replay.sh
```

可选参数：

```bash
bash scripts/weekly_replay.sh --days 30 --sources user,recharge,withdraw
bash scripts/weekly_replay.sh --start-date 2026-02-15 --end-date 2026-03-14
```

说明：
- 回放模式下，`recharge/withdraw` 使用当天窗口。
- 回放模式下跳过 `user_login` 变体，仅导出 `user_reg`。

## 6. 实时模式

```bash
bash scripts/run_realtime.sh
```

可选参数：

```bash
bash scripts/run_realtime.sh --dt 2026-03-15 --sources user,recharge,withdraw
bash scripts/run_realtime.sh --force-publish
```

默认行为：
- 每次先执行 `run_daily.sh <dt> --mode realtime --no-publish --sources ...`
- 若 `REALTIME_PUBLISH_EVERY_2H=1`，整点小时为偶数时再执行一次 `--skip-download` 发布

## 7. 回测

```bash
bash scripts/backtest.sh --dt 2026-03-05
bash scripts/backtest.sh --dt 2026-03-05 --publish
```

## 8. 用户全量回溯

```bash
bash scripts/user_full_backfill.sh --start-date 2026-02-02 --end-date 2026-03-14 --fetch
```

说明：
- `--fetch` 会调用远端导出并继续执行 `raw_import`、`normalize`、`publish.validate`。
- 不带 `--fetch` 时只会尝试处理已有本地文件（若不存在可用文件会失败）。
