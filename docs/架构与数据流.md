# 架构与数据流

## 1. 主流程（`scripts/run_daily.sh`）

实际编排入口：`src/autotag/ops/pipeline_runner.py`

执行顺序：

1. `ingest.downloader`（`--skip-download` 时跳过）
2. `load.raw_import`
3. `ingest.mobile_sync`（`ENABLE_MOBILE_SYNC=1` 时）
4. `load.normalize`
5. `load.build_mart`
6. `model.features`
7. `model.labeling`
8. `model.views_ops`
9. `model.snapshot_daily`
10. `publish.validate`（`--no-publish` 时跳过）
11. `pytest tests/test_publish_gating.py -q`
12. `publish.snapshot`

## 2. 关键输入输出

输入：
- `data/dropbox/`
- `data/initial_csv/`

输出：
- `data/raw_files/dt=YYYY-MM-DD/<source>/*.csv`
- `data/manifests/dt=YYYY-MM-DD/manifest.json`
- `data/db/serving.duckdb`
- `data/db/metabase.duckdb`
- `logs/daily/dt=YYYY-MM-DD.log`

## 3. 下载与窗口策略（`autotag.ingest.downloader`）

- `daily`
  - `user_reg_daily` + `user_login_daily`
  - `recharge/withdraw`：`D-2 ~ D` 三天窗口
  - `bet/bonus`：当天窗口
- `replay`
  - `user_reg_daily`（跳过 `user_login`）
  - `recharge/withdraw`：当天窗口
  - `bet/bonus`：当天窗口
- `realtime`
  - 各源使用最近 `REALTIME_WINDOW_MINUTES` 分钟窗口

## 4. 运行保护与可观测性

- 单实例锁：`.locks/run_daily.lock`（flock）或 `.locks/run_daily.pid`
- 运行历史：`ops_config.run_history`
- 下载状态拆分文件：`logs/daily/source_status_*`、`source_success_*`、`source_fail_*`

## 5. 发布与门禁

- 门禁逻辑：`src/autotag/publish/validate.py`
- 额外门禁测试：`tests/test_publish_gating.py`
- 发布实现：`src/autotag/publish/snapshot.py`
- 发布策略：`src/autotag/publish/atomic_publish.md`

`publish.snapshot` 先写临时库再 `os.replace` 原子替换；失败不覆盖旧库。

## 6. `--skip-download` 的链路影响

当使用 `--skip-download`：
- 跳过 `ingest.downloader`
- 自动设置：
  - `ENABLE_LOGIN_FRESHNESS_GATE=0`
  - `ENABLE_STATUS_DRIFT_GATE=0`
- 其余 `load/model/publish` 步骤照常执行
