# 功能说明

## 1. `--skip-download`

命令：

```bash
bash scripts/run_daily.sh 2026-03-14 --skip-download
```

行为：
- 跳过远端抓取
- 继续执行 `raw_import -> mobile_sync(可选) -> normalize -> build_mart -> features -> labeling -> views_ops -> snapshot_daily -> validate -> pytest -> snapshot`
- 自动关闭登录新鲜度和订单状态漂移两个门禁

## 2. 并发 API 抓取

环境变量：

```bash
API_MAX_CONCURRENT=3
```

说明：
- `downloader` 使用 `ThreadPoolExecutor` 并发抓取任务变体。
- 同一 `task_name` 使用锁串行提交，避免后端任务列表返回串扰。
- 核心源失败会中断流程；可选源失败仅记录并继续。

## 3. Token 缓存与登录重试

实现位置：
- `src/autotag/ingest/token_cache.py`
- `src/autotag/ingest/downloader.py`

当前行为：
- Token 缓存目录：`~/.autotag/cache/`
- 文件名按 `base_url|username` 命名空间哈希区分（`api_token_<digest>.json`）
- 默认 TTL：48 小时
- 登录重试：最多 3 次，指数退避（2s, 4s, 8s）

常用排查：

```bash
ls -la ~/.autotag/cache/
rm -f ~/.autotag/cache/api_token*.json
```

## 4. ID 类型口径

当前 `stg/mart` SQL 中，ID 字段按字符串处理（VARCHAR 语义），不再依赖 BIGINT 转换。

检查命令：

```bash
rg -n "AS BIGINT|TRY_CAST\(.*BIGINT|BIGINT" src/autotag/db/sql/stg src/autotag/db/sql/mart
```

## 5. 发布门禁要点

门禁入口：`autotag.publish.validate`

默认检查：
- `ops."用户状态总览"` 非空
- 关键列非空（用户ID/用户状态）
- 最大统计日期不晚于运行日期
- bonus 汇总一致性
- 登录新鲜度和状态漂移（可通过 env 开关控制）

## 6. 用户全量回溯脚本边界

`scripts/user_full_backfill.sh --fetch` 当前会执行：

1. `downloader --sources user --user-range-start/--user-range-end --fetch`
2. `load.raw_import --dt <end_date>`
3. `load.normalize --dt <end_date>`
4. `publish.validate --dt <end_date>`

不会执行 `build_mart/features/labeling/views_ops/snapshot`，如需完整链路请额外调用 `run_daily.sh` 或对应模块。
